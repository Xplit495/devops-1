## Security

### Secrets
- NEVER commit: `.env`, `*.pem`, `credentials.json`, `*.key`, API keys, tokens
- Secrets live in: `.env` (gitignored) or Vaultwarden
- Scan before commit: `git diff --staged | grep -E '(password|secret|token|key|api)'`

### Dependencies
- Audit before adding: check maintenance status, known CVEs, supply chain
- Pin versions — no floating `latest` / `@latest` tags
- Prefer well-maintained, minimal-dependency libraries

### Code
- No `eval()`, no `shell=True` with user input, no unsafe deserialization
- Validate all inputs at boundaries (API, CLI, form)
- Least privilege for DB connections, API tokens, file access
- No hardcoded credentials — ever

### Network & Exposure
- No direct exposure to the internet — use Cloudflare Tunnel, VPN, or reverse proxy
- Default-deny firewall: allow only required ports
- Internal services communicate via `.ha` hostnames or 192.168.x.x, never public IPs
- `chattr +i` on critical config files where appropriate (resolv.conf, wireguard)
